## 📖 Описание

GateRunner — минималистичная 2D «бегалка» (vertical runner) с управлением через тач/мышь. Игрок управляет отрядом юнитов, проходя через ворота (+1/-1 к количеству), уклоняясь от врагов и автоматически отстреливаясь. Все игровые объекты движутся сверху вниз. Игрок зафиксирован в нижней части экрана и перемещается только по горизонтали.

## 🕹 Игровой функционал

### Жанр и механика
- **Vertical runner** с механикой изменения размера отряда
- Игрок управляет кругом, внутри которого отображается количество юнитов
- Размер игрока динамически зависит от числа юнитов: `size = 20 + units * 2`

### Управление
- **Тач/мышь** — горизонтальное перемещение игрока
- Зона управления: нижние 15% экрана (`pointermove` отслеживается только там)
- Позиция игрока: фиксированная по Y — `canvas.height * 0.95` (5% от нижнего края)
- Клэмпинг по горизонтали: `clamp(x, radius + PADDING, canvasWidth - radius - PADDING)`
- Линейная интерполяция (lerp) для плавного движения к точке касания
- Поддержка `devicePixelRatio` для чёткой отрисовки на Retina-экранах

### Игровые объекты

#### Игрок (`src/entities/Player.js`)
- Визуал: белый круг с обводкой, число юнитов в центре
- Размер зависит от `units` (20 + units × 2 пикселей)
- Вертикальная позиция фиксирована: `y = canvas.height * 0.95`
- Вспышка (мерцание) при получении урона
- Пересчёт позиции при `resize` окна

#### Ворота (`src/entities/Gate.js`)
- Три визуальные зоны:
  - **Левый столб** — нейтральный серый `#4a4a4a`, ширина 18px
  - **Зона эффекта** — зелёная `#22c55e` для +1, красная `#ef4444` для -1
  - **Правый столб** — нейтральный серый `#4a4a4a`, ширина 18px
- Текст `+1` / `-1` по центру зоны эффекта
- Движение строго сверху вниз (`y += speed * dt`)
- При проходе через зону эффекта: `units += effectValue`
- Флаг `passed` предотвращает повторное срабатывание
- Удаляются после выхода за нижний край экрана

#### Враги (`src/entities/Enemy.js`)
- Визуал: серый ромб (алмаз)
- Параметры: `hp` (1–2), `damage` (1–2), `size` (15px)
- Горизонтальное перемещение с рандомной сменой направления
- Движение сверху вниз вместе с общим потоком
- При касании игрока: `units -= damage`, если `units <= 0` → поражение
- Вспышка при получении урона от пули
- Индикатор `-N` урона, если `damage > 1`

#### Пули (`src/entities/Bullet.js`)
- Автострельба каждые 0.4с, если на экране есть враги
- Визуал: жёлтая точка с оранжевой обводкой
- Движение строго вверх (`y -= speed * dt`)
- При попадании во врага: `enemy.hp -= 1`, при `hp <= 0` → удаление врага
- Удаляются при выходе за верхний край экрана

### Условия победы и поражения
- **Победа**: пройдены все ворота уровня (`gatesPassed / totalGates >= 1`)
- **Поражение**: `units <= 0` (от столкновения с врагом или красными воротами)
- После завершения уровня — экран результата с кнопкой «В МЕНЮ»

### Структура уровней
- Сетка выбора: 5×2, активен только уровень 1 (MVP)
- Уровень генерируется процедурно: ворота и враги появляются сверху с интервалом
- Объекты спавнятся за верхним краем экрана и движутся вниз

### Состояния приложения
```
menu → select → confirm → playing → result → menu
```
| Состояние | Сцена | Описание |
|-----------|-------|----------|
| `menu` | MenuScene | Главное меню с кнопкой «ИГРАТЬ» |
| `select` | LevelSelectScene | Сетка 5×2 выбора уровня |
| `confirm` | ConfirmScene | Подтверждение запуска |
| `playing` | GameScene | Игровой процесс |
| `won` | ResultScene | Экран победы |
| `lost` | ResultScene | Экран поражения |

## 🔧 API сцен и сущностей

### GameLoop (`src/core/GameLoop.js`)
```
new GameLoop(canvas)          — создаёт цикл с canvas
.onUpdate(callback)           — callback(dt) вызывается каждый fixed-шаг
.onRender(callback)           — callback(ctx, width, height) каждый кадр
.start() / .stop()            — управление циклом
```
Автоматически обрабатывает `resize` с `devicePixelRatio`.

### SceneManager (`src/core/SceneManager.js`)
```
.push(scene)                  — добавить сцену на стек
.pop()                        — убрать верхнюю сцену
.replace(scene)               — заменить верхнюю сцену
.clearAndPush(scene)          — очистить стек и добавить сцену
.current                      — текущая (верхняя) сцена
```
Сцены должны реализовывать: `onEnter()`, `onExit()`, `update(dt)`, `render(ctx, w, h)`, `onPointerDown(x, y)`, `onPointerMove(x, y)`, `onPointerUp()`.

### State (`src/core/State.js`)
```
State.gameState               — 'menu' | 'select' | 'playing' | 'won' | 'lost'
State.units                   — текущее количество юнитов
State.levelProgress           — прогресс уровня (0 → 1)
State.selectedLevel           — выбранный номер уровня
State.lastResult              — 'won' | 'lost' | null
State.reset()                 — полный сброс состояния
State.resetForGame()          — сброс для начала игры
```

### Система коллизий
- **Gate ↔ Player**: AABB по Y-band + проверка нахождения X в зоне эффекта
- **Enemy ↔ Player**: круговая коллизия (distance < playerRadius + enemySize × 0.7)
- **Bullet ↔ Enemy**: круговая коллизия (distance < bulletRadius + enemySize × 0.7)

### Игровой цикл
```
requestAnimationFrame → calculate deltaTime → fixed timestep (1/60s):
  1. update(dt):
     - LevelGenerator.spawn() — создать новые объекты за верхним краем
     - Gate.update(dt) — движение вниз
     - Enemy.update(dt) — горизонтальное + движение вниз
     - Bullet.update(dt) — движение вверх
     - Player.update(dt) — lerp по X
     - Коллизии: gate↔player, enemy↔player, bullet↔enemy
     - Проверка победы/поражения
  2. render(ctx, w, h):
     - Чёрный фон (#000000)
     - Ворота, враги, пули, игрок, HUD
```

## 🚀 Запуск и сборка

### Установка
```bash
npm install --legacy-peer-deps
```

### Разработка
```bash
npm run dev
```
Откроется dev-сервер на `http://localhost:3000` (доступен по сети: `http://192.168.x.x:3000`).

### Сборка
```bash
npm run build
```
Генерирует `dist/` с PWA-манифестом и service worker.

### Линтинг
```bash
npx eslint src/
npx prettier --check src/
```

### Тестирование на мобильных
1. Запустите `npm run dev` на ПК
2. Убедитесь, что телефон и ПК в одной Wi-Fi сети
3. Откройте в браузере телефона: `http://192.168.x.x:3000`
4. Для установки как PWA:
   - **Android (Chrome)**: `⋮ → Добавить на главный экран`
   - **iOS (Safari)**: `Поделиться → На экран «Домой»`

## 📌 Правила разработки

1. Никаких внешних ассетов — вся графика через Canvas API (`arc`, `rect`, `lineTo`)
2. Чёрный фон всегда: `#000000`
3. Один тач: `pointerdown/move/up` с `event.preventDefault()`
4. `devicePixelRatio` aware, логические координаты в `0..width/height`
5. `const/let`, стрелочные функции, ES6 modules
6. Комментарии только для сложной логики
