## 📖 Описание

GateRunner — минималистичная 2D «бегалка» (vertical runner) с управлением через тач/мышь. Игрок управляет отрядом юнитов, проходя через ворота с математическими операциями (+, -, *, /), уклоняясь от врагов-блоков и автоматически отстреливаясь. Все игровые объекты движутся сверху вниз. Игрок зафиксирован в нижней части экрана и перемещается только по горизонтали.

## 🕹 Игровой функционал

### Жанр и механика
- **Vertical runner** с механикой изменения размера отряда
- Игрок управляет кругом, внутри которого отображается количество юнитов
- Размер игрока динамически зависит от числа юнитов: `size = 20 + playerUnits * 0.5`
- Все арифметические операции с юнитами используют `Math.floor()` и минимум `1`

### Управление
- **Тач/мышь** — горизонтальное перемещение игрока
- Зона управления: нижние 15% экрана (`pointermove` отслеживается только там)
- Позиция игрока: фиксированная по Y — `canvas.height * 0.95` (5% от нижнего края)
- Клэмпинг по горизонтали: `clamp(x, radius + PADDING, canvasWidth - radius - PADDING)`
- Линейная интерполяция (lerp) для плавного движения к точке касания
- Поддержка `devicePixelRatio` для чёткой отрисовки на Retina-экранах

### Игровые объекты

#### Игрок (`src/entities/Player.js`)
- Визуал: белый круг с обводкой, число юнитов в центре
- Размер зависит от `playerUnits` (20 + playerUnits × 0.5 пикселей)
- Вертикальная позиция фиксирована: `y = canvas.height * 0.95`
- Вспышка (мерцание) при получении урона
- Пересчёт позиции при `resize` окна через `onResize(w, h)`

#### Ворота (`src/entities/Gate.js`)
- Спавнятся **парами** (2 ворота на шаг), заполняют 5 полос экрана
- 3 визуальные зоны на каждые ворота:
  - **Левый столб** — нейтральный серый `#4a4a4a`, ширина 12px
  - **Зона эффекта** — цвет зависит от типа
  - **Правый столб** — нейтральный серый `#4a4a4a`, ширина 12px
- 4 типа ворот (независимый рандом для каждой из пары):

| Тип | Символ | Цвет зоны | Значение | Эффект |
|-----|--------|-----------|----------|--------|
| INCREASE | `+` | `#22c55e` (зелёный) | `randomInt(2, 100)` | `units + value` |
| DECREASE | `-` | `#ef4444` (красный) | `randomInt(2, 100)` | `units - value` |
| MULTIPLY | `*` | `#3b82f6` (синий) | `randomInt(2, 10)` | `units * value` |
| DIVIDE | `/` | `#f97316` (оранжевый) | `randomInt(2, 10)` | `units / value` |

- Текст по центру зоны: `${symbol}${value}` (белый, monospace)
- Математика изменения юнитов:
  ```
  function applyGate(currentUnits, type, value) {
    let result;
    switch(type) {
      case '+': result = currentUnits + value; break;
      case '-': result = currentUnits - value; break;
      case '*': result = currentUnits * value; break;
      case '/': result = currentUnits / value; break;
    }
    return Math.max(1, Math.floor(result));
  }
  ```
- Флаг `passed = true` после первого касания, повторные проходы игнорируются
- Движение строго сверху вниз (`y += speed * dt`)
- Удаляются после выхода за нижний край экрана

#### Враги (`src/entities/Enemy.js`)
- Визуал: сетка блоков (квадратов)
- Ширина: `randomInt(1, 5)` полос, Высота: ровно 5 квадратов
- Алгоритм заполнения grid:
  - `row 0`: все ячейки `true`
  - `row 1..4`: `false` если ячейка сверху `false`, иначе `Math.random() < 0.5`
- Здоровье при спавне: `totalHP = Math.max(10, Math.floor(playerUnits + 20))`
- HP на квадрат: `Math.floor(totalHP / activeSquaresCount)`
- Визуал квадратов: fill `#b91c1c`, stroke `#7f1d1d`
- HP текст: отображается по центру всего прямоугольника врага (`textAlign: center`)
- Движение сверху вниз (`y += fallSpeed * dt`)
- Коллизия пуля → квадрат: `square.hp -= 1`, при `hp <= 0` → ячейка деактивируется
- Коллизия игрок → квадрат: `playerUnits -= Math.floor(square.hp)`
- При `activeSquares <= 0` → враг уничтожен

#### Пули (`src/entities/Bullet.js`)
- Автострельба каждые 0.4с, если на экране есть живые враги
- Визуал: жёлтая точка с оранжевой обводкой
- Движение строго вверх (`y -= speed * dt`)
- При попадании в квадрат врага: `square.hp -= 1`
- Удаляются при выходе за верхний край экрана

### Условия победы и поражения
- **Победа**: все объекты заспавнены и все ушли с экрана
- **Поражение**: `playerUnits <= 1` (от столкновения с врагом или математикой ворот)
- После завершения уровня — экран результата с кнопкой «В МЕНЮ»

### Структура уровней
- Сетка выбора: 5×2, активен только уровень 1 (MVP)
- Последовательность спавна: `[GATE_PAIR, GATE_PAIR, ENEMY]` → цикл
- Вертикальный шаг: `fixedGap = 180px` между центрами объектов
- Объекты спавнятся за верхним краем экрана и движутся вниз

### Состояния приложения
```
menu → select → confirm → playing → result → menu
```
| Состояние | Сцена | Описание |
|-----------|-------|----------|
| `menu` | MenuScene | Главное меню с кнопкой «ИГРАТЬ» |
| `select` | LevelSelectScene | Сетка 5×2 выбора уровня |
| `confirm` | ConfirmScene | Подтверждение запуска |
| `playing` | GameScene | Игровой процесс |
| `won` | ResultScene | Экран победы |
| `lost` | ResultScene | Экран поражения |

## 🔧 API

### State (`src/core/State.js`)
```
State.playerUnits               — текущее количество юнитов (минимум 1)
State.gameState                 — 'menu' | 'select' | 'playing' | 'won' | 'lost'
State.levelProgress             — прогресс уровня (0 → 1)
State.totalGates                — общее количество ворот на уровне
State.gatesPassed               — количество пройденных ворот
State.lastResult                — 'won' | 'lost' | null
applyGateMath(units, type, val) — Math.max(1, Math.floor(op)) для всех операций
```

### Spawner (`src/core/Spawner.js`)
```
new Spawner(levelNum)           — создаёт спавнер для уровня
.update(dt, canvasWidth)        — возвращает массив {type, ...params} объектов
.progress                       — прогресс уровня (0 → 1)
.finished                       — все шаги завершены
```
Последовательность: `[GATE_PAIR, GATE_PAIR, ENEMY]` → цикл. Шаг: 180px, скорость: 120px/s.

### Система коллизий
- **Gate ↔ Player**: AABB по Y-band + X в зоне эффекта
- **Bullet ↔ Enemy square**: AABB прямоугольник-прямоугольник
- **Player ↔ Enemy square**: расстояние от точки до прямоугольника < playerRadius

### Цвета ворот
| Тип | Цвет зоны | Hex |
|-----|-----------|-----|
| + (увеличение) | Зелёный | `#22c55e` |
| - (уменьшение) | Красный | `#ef4444` |
| * (умножение) | Синий | `#3b82f6` |
| / (деление) | Оранжевый | `#f97316` |
| Столбы | Серый | `#4a4a4a` |

## 🚀 Запуск и сборка

### Установка
```bash
npm install --legacy-peer-deps
```

### Разработка
```bash
npm run dev
```
Откроется dev-сервер на `http://localhost:3000` (доступен по сети: `http://192.168.x.x:3000`).

### Сборка
```bash
npm run build
```
Генерирует `dist/` с PWA-манифестом и service worker.

### Линтинг
```bash
npx eslint src/
npx prettier --check src/
```

### Тестирование на мобильных
1. Запустите `npm run dev` на ПК
2. Убедитесь, что телефон и ПК в одной Wi-Fi сети
3. Откройте в браузере телефона: `http://192.168.x.x:3000`
4. Для установки как PWA:
   - **Android (Chrome)**: `⋮ → Добавить на главный экран`
   - **iOS (Safari)**: `Поделиться → На экран «Домой»`

## 📌 Правила разработки

1. Никаких внешних ассетов — вся графика через Canvas API (`arc`, `rect`, `lineTo`)
2. Чёрный фон всегда: `#000000`
3. Один тач: `pointerdown/move/up` с `event.preventDefault()`
4. `devicePixelRatio` aware, логические координаты в `0..width/height`
5. `const/let`, стрелочные функции, ES6 modules
6. Все арифметические операции с `playerUnits` используют `Math.floor()` и минимум `1`
7. Комментарии только для сложной логики
